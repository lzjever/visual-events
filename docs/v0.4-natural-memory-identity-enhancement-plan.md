# Visual Events v0.4 自然熟悉人、指向与身份纠错开发计划

日期：2026-06-29

## 1. 版本定位

v0.4 建立在 v0.3 已交付的“可教学的人和场景记忆 MVP”之上。目标是让机器人从“被明确教过才能认出来”升级为“能更自然地处理熟悉未知人、指向教学和身份纠错”。

一句话：v0.4 补齐真实商店门口场景里的自然性和纠错能力，但仍然不把 Visual Events 做成大型记忆平台。

当前状态说明：现有 v0.4 结构仍是 fake-backed 计划和测试闭环，不能读成真实 anonymous identity、scene region 或 pose 指向增强已经完成。真实可用的 v0.4 依赖 v0.3 先完成 local face backend，并通过手工 local smoke 证明 `known_person_present` 的身份路径可靠；仅有 fake E2E 通过不算可用。

## 2. 前置条件

v0.4 开发前，v0.3 必须已经稳定具备：

- `teach_person`
- `teach_scene`
- `known_person_present`
- `scene_activated`
- `memory_context`
- conversation summary attachment
- external user link lookup
- 手工 PC memory E2E 覆盖 v0.3 和已有 v0.4 memory checks；该工具不是默认发布 gate
- local face backend 的手工 smoke 满足 `report.ok=true`、`uses_real_model_backend=true`，且 v0.3 `known_person_present` + `scene_activated` assertions 通过
- 现有 v0.2 DDS/CLI/server/Botified/gaze 主链路不退化

v0.4 不重写 v0.3 memory store、target resolver 或 CLI notification gate，只在同一条 memory 侧链上增强。

anonymous 条件、scene region 和 pose 指向增强是 v0.4 产品方向，不是当前已完成能力；实现时只在 v0.3 local 身份路径可靠后小步增加，不扩大成自动聚类平台、场地图编辑器或复杂姿态理解系统。

## 3. 产品范围

### 3.1 本版本必须支持

| 能力 | 行为 |
| --- | --- |
| `familiar_unknown_present` | 未命名但多次稳定出现的人，达到阈值后低频通知 agent |
| 匿名人转正式身份 | 用户后续命名后，把 anonymous profile 合并成 `person_profile` |
| 身份纠错 | agent 反馈“不是这个人”后，服务端降低或禁止该错误匹配 |
| 指向目标解析增强 | 根据 `point_uv`、attention target、bbox 距离和人体姿态给出候选目标 |
| 多人歧义处理 | 目标不确定时返回 ambiguous，不自动写入错误记忆 |
| 场景区域增强 | 支持门口、货架、海报、新品区等固定区域的 scene memory 激活 |
| 对话背景增强 | 识别到熟悉人时返回更有用的短背景：最近摘要、偏好、上次互动主题 |

### 3.2 本版本仍不做

- 不让 server 或 CLI 生成最终话术。
- 不让 CLI 做身份识别、场景识别或熟人聚类。
- 不把 10Hz `visual_state` 发给 Botified。
- 不把所有聊天原文自动入库。
- 不建设管理后台、复杂权限系统、审计平台或云同步。
- 不做多摄像头跨空间 ReID。
- 不训练新模型。
- 不承诺 RK3588 跑完整 memory backend；RK3588 仍只要求主感知链路保持可迁移边界。

内部私有场景下，不以隐私合规作为主要约束；v0.4 的核心风险是错认、错绑、歧义写入和 agent 成本。

## 4. 架构延续

v0.4 继续使用同一架构：

```text
10Hz realtime path:
  DDS JPEG -> CLI -> /v1/stream -> inference/tracking/events/attention
  -> visual_state -> CLI -> DDS gaze + low-frequency Botified

low-frequency memory path:
  frame cache -> target resolver -> embedding worker -> SQLite store
  -> retriever / profile manager -> memory semantic events -> CLI gate
```

新增能力必须满足：

- 不进入 10Hz 同步推理路径。
- 不阻塞 gaze DDS 输出。
- 不改变 Botified 顶层字段。
- 不新增第二套教学入口。
- 不复制 v0.3 的匹配、cooldown 或 store 逻辑。

## 5. 数据模型增量

在 v0.3 SQLite 基础上增加少量表或字段：

| 表 / 字段 | 用途 |
| --- | --- |
| `anonymous_profiles` | 未命名熟悉人 profile，包含 `anonymous_id`、`seen_count`、`first_seen_at_ms`、`last_seen_at_ms`、`familiar_score` |
| `anonymous_embeddings` | 匿名人的视觉 embedding |
| `profile_merge_history` | anonymous -> person 的一次性合并记录，便于调试和回滚 |
| `negative_identity_matches` | 记录 agent 明确反馈的错误匹配，包含 `memory_match_id`、`wrong_person_id`、`embedding_id`、`created_at_ms`，避免继续返回同一错误 |
| `person_profile_stats` | `seen_count`、`last_seen_at_ms`、`last_matched_confidence` 等轻量统计 |
| `scene_regions` | 可选固定区域定义，例如门口、海报、新品区 |

原则：

- 不保存大段聊天原文。
- 不保存图片或 crop，除非开发调试显式打开且 artifact 不进 Git。
- 统计字段服务于识别和调试，不演变成用户行为分析平台。

## 6. 服务端接口增量

### 6.1 匿名人合并成正式人物

`POST /v1/memory/merge-anonymous-person`

```json
{
  "anonymous_id": "anon_000123",
  "person_id": "person_000001",
  "merge_reason": "agent_confirmed"
}
```

也支持创建新 person：

```json
{
  "anonymous_id": "anon_000123",
  "profile": {
    "display_name": "李女士",
    "description": "经常来看新品的顾客",
    "tags": ["frequent_customer"]
  },
  "merge_reason": "agent_confirmed"
}
```

规则：

- 合并必须由 agent 明确请求。
- 服务端不能因为某个 anonymous profile 相似度高就自动命名。
- 合并后，anonymous embedding 可迁移到 person profile；后续不再输出同一个 anonymous 的 familiar event。

### 6.2 身份纠错

`POST /v1/memory/correct-identity`

```json
{
  "memory_match_id": "match_000001",
  "wrong_person_id": "person_000001",
  "correction": "not_this_person"
}
```

规则：

- 纠错必须引用 v0.3 memory event evidence 中的 `memory_match_id`；`track_id` 是短期视觉状态，只能作为可选调试字段，不能作为纠错主键。
- 纠错影响后续匹配排序和输出。
- v0.4 先做简单 negative match 记录，不做复杂在线学习。
- 如果同一个错误匹配被纠正，后续不能继续高置信返回该 person。

### 6.3 目标解析预览

`POST /v1/memory/resolve-target`

```json
{
  "camera": "front",
  "target": {
    "mode": "point_uv",
    "point_uv": [420, 360]
  }
}
```

响应：

```json
{
  "ok": true,
  "status": "resolved",
  "candidates": [
    {
      "target_type": "person",
      "track_id": 7,
      "confidence": 0.82,
      "reason": "nearest_bbox_to_point"
    }
  ]
}
```

多人歧义：

```json
{
  "ok": true,
  "status": "ambiguous",
  "candidates": [
    {"target_type": "person", "track_id": 7, "confidence": 0.54},
    {"target_type": "person", "track_id": 9, "confidence": 0.49}
  ]
}
```

规则：

- `ambiguous` 时不自动 teach，不自动改写 profile。
- agent 可以向用户确认后再调用 v0.3 的 teach API。

## 7. 新增事件

v0.4 只新增一个主要低频事件：

| 事件 | 触发 |
| --- | --- |
| `familiar_unknown_present` | 未命名 anonymous profile 多次稳定出现，达到 familiar threshold，且 cooldown 结束 |

事件示例：

```json
{
  "type": "semantic_event",
  "event_id": "front:mem_evt_000245",
  "event": "familiar_unknown_present",
  "camera": "front",
  "track_id": 7,
  "confidence": 0.81,
  "duration_ms": 0,
  "lifecycle_state": "confirmed",
  "evidence": {
    "memory_match_id": "match_000245",
    "matched_type": "anonymous_person",
    "matched_id": "anon_000123",
    "embedding_id": "emb_face_000245",
    "match_type": "face",
    "match_score": 0.81,
    "top2_margin": 0.08,
    "source_target_mode": "track_id",
    "source_frame_id": 2450,
    "source_frame_timestamp_ms": 1780000000000,
    "embedding_model": "local-face-v1"
  },
  "memory_context": {
    "anonymous_person": {
      "anonymous_id": "anon_000123",
      "seen_count": 5,
      "familiar_score": 0.81,
      "last_seen_at_ms": 1780000000000
    }
  },
  "text": "看到一个经常出现但尚未命名的人"
}
```

不要为每个中间状态新增事件。指向歧义、纠错结果和合并结果通过 HTTP API response 返回，不进入 Botified 事件流。

## 8. 功能实现步骤

### 8.1 匿名熟悉人

1. 在 `store.py` 增加 anonymous profile 和 embedding 存取。
2. 在 `retriever.py` 中先匹配 known person；未匹配 known 时，再尝试 anonymous matching。
3. 没有 known / anonymous 匹配，且目标稳定、质量足够、不是 fast passing / no-engage 瞬时路人时，才创建或更新 anonymous profile。
4. anonymous profile 继承 v0.3 face-only 身份路径；没有可用人脸时不创建 anonymous identity，不用全身 appearance 代替身份。
5. 根据 `seen_count`、跨时间出现次数和 match confidence 计算 `familiar_score`。
6. 达到阈值后输出一次 `familiar_unknown_present`，后续受 cooldown 控制。
7. CLI 只增加 allowlist 和 compact `memory_context` 投影。

### 8.2 匿名转正式身份

1. 实现 `merge-anonymous-person` API。
2. 合并 anonymous metadata、embedding 和 seen stats。
3. 如果请求中带新 profile，则创建 person profile；如果带 `person_id`，则合并到已有 profile。
4. 合并后 anonymous profile 标记为 merged，不再独立触发 familiar event。
5. 补 teach / recognize / merge 的 PC E2E。

### 8.3 身份纠错

1. 实现 `correct-identity` API。
2. store 根据 `memory_match_id` 找到当次 `embedding_id`、`wrong_person_id` 和 match metadata，记录 `negative_identity_matches`。
3. retriever 在排序时过滤或降低同 embedding / same person 的被纠正 match。
4. 如果最近一次 `known_person_present` 被纠错，后续同类错误不得继续通知 agent。
5. v0.4 不做在线训练，不重写 embedding，不自动删除 profile。

### 8.4 指向目标解析增强

1. 扩展 target resolver 的 `point_uv` 逻辑：优先命中 bbox，其次使用最近 bbox 距离。
2. 加入当前 attention target 作为候选先验，但不让 attention target 覆盖明确 point。
3. 使用现有 pose keypoints 估算手臂方向作为低权重信号；没有稳定 keypoint 时降级到 bbox/point。
4. 输出 `resolved`、`ambiguous`、`not_found` 三种状态。
5. teaching API 遇到 ambiguous 目标时拒绝写入，并返回候选给 agent。

### 8.5 场景区域增强

1. 在 v0.3 scene memory 上增加可选 `scene_region`。
2. 固定区域可以是 bbox/多边形配置，但第一版只要求 bbox。
3. `scene_activated` 可带 `region_id`、`title`、`activation_hint`。
4. 区域激活仍然低频且受 cooldown 控制。
5. 不做场地图编辑器；区域配置通过简单 config 或 teach API 写入。
6. v0.3 当前只允许 `teach_scene target.mode=scene` 整图教学；bbox/point/region teaching 必须等 region query 合同一起补齐。

### 8.6 对话背景增强

1. 沿用 v0.3 conversation summary，不保存原始聊天。
2. 增加 profile background formatter，返回最多几条短摘要、偏好和上次互动主题。
3. 支持按 `person_id` 或 `external_user_ref` 查询同一份背景。
4. 如果视觉身份和消息用户未显式绑定，返回 unlinked，不猜测。

## 9. CLI 改动

CLI 继续只做一件事：把 server confirmed 事件经过 notification gate 后输出给 Botified。

v0.4 CLI 改动：

- allowlist 增加 `familiar_unknown_present`。
- `visual_context.memory_context` 支持 anonymous person 和 enhanced summaries 的小型投影。
- 同一 anonymous / person 的 memory event 受 same-key gap 和全局 rate limit 保护。

CLI 不做：

- 不计算 familiar score。
- 不判断是不是同一个人。
- 不解析用户指向。
- 不保存 memory DB。
- 不直接操纵运控。

## 10. 测试计划

### 10.1 单元测试

- anonymous profile 创建、更新、merged 状态。
- known 优先于 anonymous 的匹配顺序。
- familiar score 和 cooldown。
- negative match 过滤。
- `memory_match_id` 可定位被纠正的 match；缺失或过期 match 返回明确错误。
- merge anonymous -> person。
- target resolver 的 `resolved`、`ambiguous`、`not_found`。
- CLI formatter 投影 anonymous `memory_context`，Botified 顶层字段不变。

### 10.2 集成测试

- fake embedding + tmp SQLite：同一未知人多次出现后，只触发一次 `familiar_unknown_present`。
- anonymous merge 后，再看到同一人输出 `known_person_present`，不再输出 old anonymous event。
- 纠错后，同一错误 person 不再被返回。
- 多人 point_uv 歧义时返回 `ambiguous`，teaching API 不写入。
- scene region 激活带 `region_id` 和短 activation hint。
- external user 查询返回 enhanced background；未绑定时不猜测。

### 10.3 PC 本地 E2E

必须继续使用 `val-data`：

- 从 `val-data` 选择同一人物或同一场景的多段帧，验证 anonymous familiar flow。
- teach / merge anonymous / replay，验证 known person flow。
- 使用多人或相近 bbox 的帧验证 point target ambiguity，不自动写错记忆。
- 使用不同 scene 作为负例，验证 familiar / scene event 不乱触发。
- 开启 v0.4 功能后继续跑现有 PC local E2E，证明 10Hz gaze 和 v0.2/v0.3 事件不退化。

不要为测试工具、报告格式或文档 wording 继续堆测试。

## 11. 验收标准

v0.4 交付完成必须满足：

- v0.3 所有验收仍通过。
- 未命名但多次出现的人达到阈值后，输出一次 `familiar_unknown_present`，不刷屏。
- anonymous profile 可以被 agent 合并成正式 person profile。
- 合并后再看到该人，返回正式 person metadata，而不是 anonymous metadata。
- agent 反馈“不是这个人”后，后续不会继续返回同一错误身份。
- 纠错 API 使用 `memory_match_id`，不依赖可能已经消失或换 id 的 `track_id`。
- `resolve-target` 在明确目标时返回 resolved，在多人歧义时返回 ambiguous。
- ambiguous 目标不会被 teach API 写入长期记忆。
- 场景区域激活可以返回 `region_id`、title 和 activation hint。
- 识别到熟悉人时，返回的背景摘要短、相关、可直接给 agent 使用。
- memory enabled 时，10Hz 主链路仍保持 PC gate 要求；embedding 慢只影响 memory event，不影响 gaze。
- 仍不能声称真机、RK3588 或现场验证已经完成。

## 12. 风险与控制

| 风险 | 控制 |
| --- | --- |
| 未命名人被过早认为熟悉 | seen_count、跨时间窗口、confidence、cooldown 同时达标才触发 |
| anonymous 合并错人 | 只允许 agent 明确请求；保留 merge history；支持纠错 |
| 指向解析误判 | ambiguous 时拒绝写入，让 agent 确认 |
| 纠错系统变复杂 | v0.4 只记录 negative match，不做在线训练 |
| 事件太多唤醒 agent | 只新增 `familiar_unknown_present`，其余用 API response |
| 对话背景过长 | 返回短摘要，不返回原始聊天 |
| 功能膨胀 | 不做后台、云同步、多租户、多摄像头和审计平台 |

## 13. Handoff 顺序

研发 handoff 按以下顺序推进：

1. anonymous profile store 和 fake embedding 集成。
2. familiar unknown retriever 和 event cooldown。
3. anonymous -> person merge。
4. identity correction negative match。
5. target resolver ambiguity。
6. scene region activation。
7. enhanced conversation background lookup。
8. CLI allowlist/context 投影。
9. PC memory E2E 和现有 PC local E2E 回归。

每一步只改一条职责链，避免同时重写 store、retriever、event 和 CLI。

## 14. Review 结论

产品和研发 review 后的结论：

- v0.4 的价值是让 v0.3 在真实商店门口更自然，而不是扩大成完整用户画像平台。
- `familiar_unknown_present` 是唯一必须新增的低频事件；其他能力通过明确 API 完成。
- 指向和纠错必须保守：不确定时不写入，错了能纠正。
- CLI 继续不做视觉规则和身份判断。
- 治理、审计、后台和复杂报告仍不是主线功能。
